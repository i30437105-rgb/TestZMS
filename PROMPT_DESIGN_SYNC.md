## Правила работы:

1. **Ничего не делать без указания заказчика.** Ждёшь, какой блок дорабатывать.
2. **Работаем пошагово** — один блок за раз. Не переходить к следующему без утверждения.
3. **Лендинг = первоисточник стиля.** Все дизайн-решения сверяются с лендингом.
4. **НЕ менять то, о чём не просили.** Это критически важно.
5. **Показывать результат после каждого изменения** — заказчик проверяет визуально.
6. **Фиксировать каждое утверждённое решение** в `DESIGN_SOURCE.md` → раздел "Утверждённые решения".
7. **Спрашивать, если неоднозначно.** Лучше уточнить, чем додумать.

## Процесс работы:

```
Заказчик говорит что менять → Ты делаешь → Заказчик проверяет → Утверждает или корректирует → Фиксируешь решение в DESIGN_SOURCE.md → Следующий блок
```

## Что НЕ делать:

- ❌ Не делать "улучшения" без запроса
- ❌ Не менять цветовые схемы самостоятельно
- ❌ Не трогать функциональность (только визуал)
- ❌ Не переписывать компоненты целиком
- ❌ Не принимать дизайн-решения за заказчика
